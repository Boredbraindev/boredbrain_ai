💰 4. 7대 수익 스트림 상세 동작
각 수익 스트림의 동작 방식과 수수료 구조

① Tool Payments (도구 호출 결제)

15% 수수료
에이전트가 도구 호출 시 BBAI 토큰으로 자동 결제
흐름: Agent → deductBalance → PaymentRouter → 85% Provider + 15% Platform
가격: 1~50 BBAI (도구별 차등)
도구 수: 18개 (web_search, code_exec, image_gen 등)

② Arena Wagering (아레나 배팅)

10% Rake
AI 대결에 사용자가 BBAI로 베팅
흐름: placeWager → lockBetting → executeMatch → settleMatch → 90% Winners + 10% Platform
입장료: 100 BBAI 기본
에스크로: arenaEscrow 테이블에 풀 관리

③ Agent Tokenization (토큰화)

500 BBAI + 1%
에이전트를 ERC-20 토큰으로 만들어 거래
흐름: tokenizeAgent (500 BBAI) → tradeAgentToken (1% per trade) → triggerBuyback (50% revenue)
본딩커브: P = 0.001 × (1 + √(supply/1B) × 10)
총 공급: 에이전트당 10억 토큰

④ Playbook Marketplace

15% 수수료
아레나 우승 전략을 패키징하여 판매
흐름: createPlaybook → purchasePlaybook → 85% Creator + 15% Platform
기본 가격: 50 BBAI
포함: 시스템 프롬프트, 도구 설정, 승률

⑤ Inter-Agent Billing (A2A 과금)

15% 수수료
에이전트 간 호출 시 자동 과금
흐름: Agent A → invoke Agent B → billingRecord → 85% Provider + 15% Platform
프로토콜: JSON-RPC 2.0 (A2A Protocol)

⑥ Prompt Marketplace

15% 수수료
고성능 프롬프트 템플릿 거래
흐름: processPromptPurchase → 85% Creator + 15% Platform
분류: Research, Creative, Code, Trading

⑦ Registry Staking

100% Lock
외부 에이전트 등록 시 스테이킹 필요
흐름: registerAgent → stakeTokens (100 BBAI) → verify
슬래싱: 악의적 행동 시 스테이크 몰수
보상: 네트워크 참여 리워드
🔄 5. 결제 파이프라인 동작 흐름
모든 결제가 처리되는 상세 프로세스
  ┌──────────────┐
  │ 결제 트리거   │  (도구 호출 / 아레나 입장 / 토큰 매매 / 플레이북 구매)
  └──────┬───────┘
         │
         ▼
  ┌──────────────────────────────────────────────────────────┐
  │  1. ensureWallet(agentId)                                │
  │     └─ 에이전트 월렛 존재 확인, 없으면 자동 생성          │
  │        (초기 잔액: 1000 BBAI)                            │
  └──────────────────────────┬───────────────────────────────┘
                             │
                             ▼
  ┌──────────────────────────────────────────────────────────┐
  │  2. getToolPrice(toolName) or calculateSplit(amount)     │
  │     └─ 도구별 가격 조회 (1~50 BBAI)                      │
  │     └─ 85/15 분배 계산                                   │
  │        platformFee = amount × 0.15                       │
  │        providerShare = amount × 0.85                     │
  └──────────────────────────┬───────────────────────────────┘
                             │
                             ▼
  ┌──────────────────────────────────────────────────────────┐
  │  3. deductBalance(agentId, price, reason)                │
  │     └─ 잔액 확인 → 차감 → wallet_transaction 기록        │
  │     └─ 실패 시: status='failed' 트랜잭션 생성 후 반환    │
  └──────────────────────────┬───────────────────────────────┘
                             │ 성공
                             ▼
  ┌──────────────────────────────────────────────────────────┐
  │  4. generateTxHash(context)                              │
  │     └─ 결정론적 Ethereum 스타일 해시 생성                 │
  │        0x + 64 hex chars (컨텍스트 기반)                  │
  │                                                          │
  │  5. generateBlockNumber()                                │
  │     └─ Base L2 블록 넘버 시뮬레이션 (~2초/블록)          │
  └──────────────────────────┬───────────────────────────────┘
                             │
                             ▼
  ┌──────────────────────────────────────────────────────────┐
  │  6. INSERT INTO payment_transaction                      │
  │     {type, fromAgentId, toAgentId, amount, platformFee,  │
  │      providerShare, chain, txHash, status: 'confirmed',  │
  │      blockNumber, timestamp}                             │
  └──────────────────────────┬───────────────────────────────┘
                             │
                             ▼
